# Blender Python 脚本

一个基本的 Blender Python 脚本（使用 bpy 模块）通常遵循这样的结构：导入必要的模块，通过 API 访问 Blender 数据，并为特定任务定义类或函数。而且 Blender 里面的一切都是操作，都是一个 Python 函数。 所以基本上，如果可以用 UI 做到，就应该可以用脚本写出来。

## 核心 API 模组


|关键模块|概念|实例|
|:---:|:---|:---|
|bpy|主要模块，用于提供连接 Python API，对数据、运算符和用户界面的访问|import bpy|
|bpy.data|用于读取或修改内存中任何对象或数据块的中心枢纽|bpy.data.objects['Cube']|
|bpy.ops|用于在视口中手动执行的操作。它们返回类似 {'FINISHED'} 或 {'CANCELLED'} 的结果|bpy.ops.mesh.primitive_cube_add()|
|bpy.props|与 bpy.types 结合使用，用于定义插件工具的变量|bpy.props.StringProperty(update=updateNode)|
|bpy.utils|使用 register() 和 unregister() 将 Python 代码转换为可安装的插件所必需的|bpy.utils.register_class(MainPanel)|


# bpy.data

bpy.data 是存取当前 .blend 档案中所有内部数据的主要入口。 它就像是一个大型资料库，存放着场景中的所有「数据块」（Data-blocks），例如模型、材质、灯光和动画。  

## 核心功能与常用函式

bpy.data 下包含了多个集合（Collections），每个集合对应不同类型的资源： 


- 物体 (Objects): bpy.data.objects
   - 存取场景中的所有物体（如 Cube, Camera, Lamp）。
   - 范例：bpy.data.objects["Cube"] 可以取得名为 "Cube" 的物体。
- 网格 (Meshes): bpy.data.meshes
  - 存取几何数据。 你可以透过 new() 函数来手动创建新的网格。
l
- 材质 (Materials): bpy.data.materials
   - 管理所有的材质球，可进行属性修改或删除。
- 图像与纹理: bpy.data.images
   - 存取载入到档案中的外部图片或内部的渲染图。
- 场景 (Scenes): bpy.data.scenes
   - 存取档案中的不同场景设定。  

# bpy.ops

bpy.ops 是 核心模组，专门用来执行各类「运算子（Operators）」。 简单说，它就像是用程式码去点击介面上的按钮或选单指令。  

## 核心功能与常用函式

运算子按功能分类在不同的子模组下，常见的包括：

 - bpy.ops.object：处理物体的操作，如添加、删除、变形或套用修改器。
- bpy.ops.mesh：针对网格编辑模式的操作，如挤出（Extrude）、细分（Subdivide）或合并顶点。
- bpy.ops.render：启动渲染程序。
- bpy.ops.wm：视窗管理器操作，用于开启文件、储存专案或跳出警告讯息。
- bpy.ops.transform：移动、旋转及缩放等基础空间操作。  


# bpy.props

bpy.props 是一个用于定义自订属性的模组。 它的主要作用是扩展内部数据结构，让开发者可以在 Operator（运算器）、Panel（面板）或数据类型（如 Scene 或 Object）中添加自订的设定项。  

## 主要用途

 - 扩展数据：为  bpy.types.Scene、bpy.types.Object 等类别添加新的栏位。
 - UI 交互：在自订面板或选单中显示输入框（如滑动条、勾选框）。
 - 回呼功能：透过 update 参数，在属性值改变时自动执行特定的 Python 函式。  

## 常用属性类型

最常见的类型包括：

|类型|用途|
|:---:|:---:|
|BoolProperty|复选框（True/False）|
|IntProperty|整数输入|
|FloatProperty|浮点数输入（可设定精度与单位）|
|StringProperty|文字输入|
|EnumProperty|下拉式选单或枚举选项|
|PointerProperty|引用另一个 PropertyGroup 或特定 ID 数据
|CollectionProperty|属性集合（类似清单，可储存多个自订物件）|


# bpy.utils

bpy.utils 是 Blender Python API 中的一个子模组，主要包含与 Blender 特定功能相关、但与内部数据（如 bpy.data）不直接相关的辅助函式。  

这个模组通常用于注册插件、处理路径、管理预设值以及单位转换。

## 核心功能与常用函式

 - 注册与注销 (Registration)，这是开发插件最常用的功能，用于将自定义的类别（如 Operator 或 Panel）整合到 Blender 介面中。
   - bpy.utils.register_class(cls): 注册一个类别。
   - bpy.utils.unregister_class(cls): 注销一个类别。
   - bpy.utils.register_tool(tool_cls): 注册自定义工具。
- 路径管理 (Path Handling)，用于寻找资源路径或处理当前档案的路径，bpy.utils.resource_path(type, major, minor): 返回指定资源（如 USER 或 SYSTEM）的路径。
    - bpy.utils.script_paths(): 返回所有脚本搜寻路径的列表。
    - bpy.utils.blend_paths(absolute=False): 返回当前 .blend 档案所引用的所有外部档案路径。
 - 子模组 (Submodules)
    - bpy.utils.units: 处理单位转换（如公制与英制之间的转换）。
    - bpy.utils.previews: 管理插件中的自定义图示或缩图预览。
 - 其他公用工具
    - bpy.utils.execfile(filepath): 将档案作为 Python 脚本执行。
    - bpy.utils.escape_identifier(string): 将字串转换为合法的 Python 标识符。

## 注册 Class

```py
import bpy

# 遵循 PEP8 类名风格指南

class SimpleOperator(bpy.types.Operator):
  bl_idname = "object.simple_operator" # 唯一的内部名称（小写，以下划线分隔）
  bl_label = "简单运算符" # 易于理解的名称
  bl_options = {'REGISTER', 'UNDO'} # 撤销等选项

  def execute(self, context): 
    # 运算符的主要逻辑写在这里，此示例仅向控制台打印一条消息并添加一个立方体
    print(f"Hello Blender! Adding a cube at {context.scene.cursor.location}")
    bpy.ops.mesh.primitive_cube_add(location=context.scene.cursor.location)
    return {'FINISHED'} # 告诉 Blender 操作已完成 成功

# --- 注册 --- 此部分在 Blender 中执行脚本时运行
  def register():
    bpy.utils.register_class(SimpleOperator)
    print("SimpleOperator 已注册")

  def unregister():
    bpy.utils.unregister_class(SimpleOperator)
    print("SimpleOperator 已取消注册")

# 允许脚本直接在 Blender 的脚本工作区运行
if __name__ == "__main__":
  register()
```

## 添加立方体

- 添加立方体，使用 bpy.ops.mesh.primitive_cube_add() 运算符。此脚本演示了如何在指定位置添加立方体并调整其大小。


```py
import bpy

def add_cube_example():

  # 向当前 Blender 场景添加一个具有特定属性的立方体。
  # 检查用户是否处于正确的模式（对象模式最适合添加对象）
  if bpy.context.mode != 'OBJECT':

    bpy.ops.object.mode_set(mode='OBJECT')

    # 向场景添加一个新的立方体图元
    # 可根据需要调整参数：
    # size: 初始缩放比例（Blender 单位）
    # enter_editmode: False 表示创建后保持在对象模式
    # align: 新对象的对齐方式（世界坐标系、视图坐标系、光标坐标系）
    # location: X、Y、Z 坐标
    # scale: X、Y、Z 缩放比例
    bpy.ops.mesh.primitive_cube_add(size=1, enter_editmode=False, 
      align='WORLD', location=(0, 0, 0), scale=(2, 1, 0.5))

    # 可选：您可以获取新创建对象的引用以对其进行进一步修改
    new_cube = bpy.context.active_object
    new_cube.name = "MyScriptedBox"
    print(f"添加的立方体名称：{new_cube.name}")

if __name__ == "__main__":
  add_cube_example()
```


## 修改 UI

 - 在 3D 視圖側邊欄（N 面板）顯示「第一個修改器」屬性的簡單腳本範例


```py
import bpy

class TEST_PT_ModifierUI(bpy.types.Panel):
  bl_label = "現有修改器控制"
  bl_idname = "TEST_PT_modifier_ui"
  bl_space_type = 'VIEW_3D'
  bl_region_type = 'UI'
  bl_category = 'MyTab'

  def draw(self, context):
    layout = self.layout
    obj = context.active_object

    if obj and obj.modifiers:
      # 取得第一個修改器作為範例
      mod = obj.modifiers[0]
      layout.label(text=f"修改器: {mod.name} ({mod.type})")
            
      # 根據修改器類型顯示對應屬性
      if mod.type == 'SUBSURF':
        layout.prop(mod, "levels", text="視圖細分")
        layout.prop(mod, "render_levels", text="渲染細分")
      elif mod.type == 'BEVEL':
        layout.prop(mod, "width", text="寬度")
        layout.prop(mod, "segments", text="段數")
      else:
        layout.label(text="未定義此類型的 UI 顯示")
    else:
      layout.label(text="請選擇含有修改器的物體")

def register():
  bpy.utils.register_class(TEST_PT_ModifierUI)

def unregister():
  bpy.utils.unregister_class(TEST_PT_ModifierUI)

if __name__ == "__main__":
  register()
```


## 创建曲线

 - 根据坐标列表创建曲线的基本脚本

```py
import bpy

# 定义路径点
coords_list = [(0,0,0), (1,1,0), (2,-1,0), (3,0,0)]

# 创建曲线数据和新的样条曲线
crv = bpy.data.curves.new('ProceduralPath', 'CURVE')
crv.dimensions = '3D'
spline = crv.splines.new(type='NURBS')

# 设置点
spline.points.add(len(coords_list) - 1)
for i, coord in enumerate(coords_list):
  spline.points[i].co = (coord[0], coord[1], coord[2], 1)

# 链接到场景
obj = bpy.data.objects.new('PathObject',  crv)

bpy.context.collection.objects.link(obj)
```
